# 🧠 AI Agent 规划能力深度笔记

规划（Planning）是 Agent 处理复杂、多步骤任务的核心大脑机制。

---

## 1. 推理技术演进

| 技术名称 | 核心思想 | 解决的问题 | 工程评价 |
| :--- | :--- | :--- | :--- |
| **CoT (思维链)** | "Let's think step by step" | 减少显式推理缺失导致的跳步错误 | **必选标配**，零成本 |
| **ToT (思维树)** | 搜索 + 评估 + 剪枝 | 解决单链推理一旦走错无法纠偏的问题 | **高质高价**，成本增加 3-5 倍 |
| **GoT (思维图)** | 推理节点复用与合并 | 解决不同路径结论无法聚合的问题 | **学术前沿**，工程落地难 |

---

## 2. 核心架构：Plan-and-Execute

这种模式将“思考”与“劳动”解耦：
1. **Planner (规划器)**：负责宏观拆解。输出：`[Task 1, Task 2, Task 3]`。
2. **Executor (执行器)**：负责微观执行。针对 `Task 1` 调用工具并返回结果。
3. **Re-planner (动态调整)**：每步执行后，重新审视剩余计划。如果 Task 1 失败或发现新信息，立即重绘地图。

---

## 3. 为什么需要规划能力？

- **误差消除**：Transformer 是预测下一个 Token 的概率模型，推理链条越长，隐性误差累积越快。
- **全局视角**：简单的 ReAct 模式是“走一步看一步”，Plan-and-Execute 则是“看准了再走”。
- **资源优化**：可以用昂贵的高性能模型做 Planner，用便宜的小模型做 Executor。

---

## 🎯 面试核心要点

- **不要只答 CoT**：要体现出从线性到树状、再到图状的思维深度。
- **成本敏感度**：一定要提到 ToT 带来的 Token 开销问题，这体现了你的工程实践经验。
- **动态性**：强调规划不是死的，**Re-planning**（重新规划）才是对抗真实世界不确定性的关键。
