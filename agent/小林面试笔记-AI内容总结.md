# 小林面试笔记 AI 专题总结

- 来源：https://xiaolinnote.com/ai/
- 访问时间：2026-04-14
- 页面性质：这是一个 **AI 面试题总览页**，核心内容不是单篇长文，而是围绕 **Agent、RAG、LLM 工具调用** 三大方向搭建的专题目录与学习框架。

## 一、页面核心信息

这页内容的重点不是直接给出所有答案，而是告诉你：

1. 现在 AI 面试里最常被问的方向有哪些；
2. 每个方向通常会从哪些层次追问；
3. 真正容易拉开差距的，不是会背概念，而是能不能讲清楚 **原理、设计取舍、工程落地问题**。

作者强调这套内容来自真实面经，写法偏“面试复盘 + 原理拆解”，目标是帮助读者：

- 不只是背答案；
- 而是建立可迁移的知识框架；
- 面试官换个问法时也能自己推导。

---

## 二、整体知识地图

页面把 AI 面试内容拆成 3 个主专题：

### 1. Agent 面试专题

重点关注：**Agent 是什么、怎么设计、怎么落地、怎么协作**。

可归纳为四层：

#### （1）概念与架构

要先讲清楚：

- Agent 和普通 LLM 调用的本质区别；
- Agent 的核心组件有哪些；
- `Workflow / Agent / Tools` 三者的定义与边界。

这部分是地基，回答不清楚，后续所有设计题都容易失分。

#### （2）设计范式

页面点名了 3 个高频范式：

- `ReAct`
- `Plan-and-Execute`
- `Reflection`

面试重点不只是“知道是什么”，更是：

- 各自解决什么问题；
- 成本和局限是什么；
- 适合什么场景；
- 项目里为什么这样选。

#### （3）工程实践

这一块更偏真实落地经验，例如：

- 复杂任务如何拆分；
- 记忆模块如何设计；
- 长短期记忆怎么存储和使用；
- 如何赋予 Agent 规划能力；
- 反思机制为什么需要、怎么实现；
- 为什么有时要“手搓 Agent”而不是直接上框架。

这类问题最能区分“了解概念”和“真正做过项目”。

#### （4）多 Agent 协作

关注点包括：

- 什么是 `Multi-Agent`；
- `Single-Agent` 和 `Multi-Agent` 怎么选；
- Agent 之间如何通信；
- 是用消息传递还是共享状态；
- 路由机制是静态规则还是 LLM 动态决策。

这是偏系统设计层面的高阶考点。

#### Agent 专题高频问题清单

- 什么是 Agent？
- Agent 与大模型的本质区别是什么？
- Agent 的基本架构有哪些核心组件？
- `Workflow / Agent / Tools` 的区别是什么？
- `ReAct / Plan-and-Execute / Reflection` 如何选型？
- 复杂任务如何拆分？
- Agent 记忆系统如何设计？
- 如何做长短期记忆与记忆压缩？
- 为什么有时不用框架而是自己实现？
- 如何设计多 Agent 的协作与动态切换？

---

### 2. RAG 面试专题

重点关注：**如何把外部知识接入大模型，并把 RAG 做成可用、可评估、可持续更新的系统**。

同样可以拆成四层：

#### （1）基础原理

要先说清楚：

- 什么是 RAG；
- 它主要解决大模型哪些问题；
- RAG 和微调各适合什么场景；
- 两者的优劣分别是什么。

这部分考查候选人对“知识增强路线”的基本判断力。

#### （2）索引构建

这是搭系统前的基础工程，主要包括：

- 文档怎么切割（Chunking）；
- 粒度怎么定；
- 语义被切断了怎么办；
- Embedding 是什么；
- Embedding 模型怎么选择和评估；
- 向量数据库怎么选型、怎么使用。

这一层看起来基础，但非常容易被深挖。

#### （3）检索优化

这是 RAG 面试里最容易拉开差距的部分，核心包括：

- 向量检索 vs 关键词检索；
- Query Rewrite 的作用和做法；
- 多路召回怎么设计；
- 常见检索优化策略有哪些；
- 更复杂的 RAG 形态，如 `Self-RAG`、`Corrective RAG`；
- 什么情况下要引入图数据库。

换句话说，面试官往往不是只想听“会做检索”，而是想听“如何把检索做好”。

#### （4）生产落地

偏工程问题：

- 如何规避幻觉；
- 如何量化 RAG 效果；
- 知识库如何动态持续更新；
- 实际落地最难的地方是什么。

这部分最能体现你是否真正经历过线上系统建设。

#### RAG 专题高频问题清单

- 什么是完整的 RAG 工作流程？
- RAG 主要解决哪些问题？
- RAG 和微调如何取舍？
- 文档切割策略怎么设计？
- 如何降低语义切断的损失？
- Embedding 的原理、选型与评估怎么做？
- 向量数据库如何选型？
- 查询进入 RAG 后系统如何运行？
- 如何做 Query Rewrite 和多路召回？
- 如何优化召回与排序效果？
- 如何抑制幻觉、评估效果、持续更新知识库？

### 3. LLM 工具调用面试专题

重点关注：**让模型从“会说话”升级为“会干活”**。

页面认为这是近阶段越来越高频的方向，核心概念包括：

- `Function Calling`
- `MCP`
- `Agent Skill`
- `A2A`
- 通信协议
- LLM 网关

#### （1）Function Calling 基础

要回答清楚：

- 什么是 `Function Calling`；
- 模型如何学会调用工具；
- 这项能力是怎么训练出来的；
- 模型负责决策还是执行；
- 输出格式大概是什么样。

这部分是整个工具调用体系的地基。

#### （2）MCP 协议

需要掌握：

- 什么是 `MCP`（模型上下文协议）；
- 它的核心内容是什么；
- 它由哪些部分组成。

理解了 MCP，才有能力和其他工具调用方案做对比。

#### （3）FC 与 MCP 的选型

页面强调一个现实问题：

- `Function Calling` 和 `MCP` 都属于工具调用；
- 但两者并不等价；
- 不同场景下适合的方案不一样；
- 有些推理模型不支持 MCP，也要知道原因。

这里考查的是“概念区分 + 架构选型能力”。

#### （4）Skill / A2A / 协议层 / 网关层

更高阶的问题包括：

- `Skill` 是什么；
- `Skill` 和 `MCP` 的区别；
- `Function Calling / Skill / MCP` 三者的关系；
- 什么是 `A2A`，它和 `MCP` 的区别是什么；
- MCP 常用什么通信方式；
- `SSE / WebSocket / WebRTC` 的差异和适用场景；
- LLM 网关解决了什么工程问题。

这部分从“工具调用”继续往下延伸到了 **协议、协作和工程治理**。

#### 工具调用专题高频问题清单

- 什么是 Function Calling？原理是什么？
- LLM 如何学会调用外部工具？
- Function Calling 能力如何训练？
- 什么是 MCP？其架构和组成是什么？
- MCP 与 Function Calling 如何对比选型？
- 为什么部分模型不支持 MCP？
- Skill 的定义和作用是什么？
- Skill、MCP、Function Calling 三者是什么关系？
- 什么是 A2A？它和 MCP 的差异是什么？
- `SSE / WebSocket / WebRTC` 各自适合什么场景？
- LLM 网关在生产环境中解决什么问题？

## 三、这页内容的共同主线

这三个专题虽然方向不同，但背后的考查逻辑非常一致：

### 1. 不只考“概念”，更考“边界”

比如：

- Agent 和 Workflow 的边界；
- RAG 和 Fine-tuning 的边界；
- Function Calling 和 MCP 的边界。

真正高质量回答，往往不是“定义背出来”，而是能解释 **为什么要区分、什么时候选哪个**。

### 2. 不只考“原理”，更考“取舍”

比如：

- 为什么选某种 Agent 设计范式；
- 为什么用某种切块策略；
- 为什么选某类检索方案；
- 为什么某个场景用 FC 而不用 MCP。

这类问题考查的是工程判断力。

### 3. 不只考“做法”，更考“落地”

页面多次提到的关键词其实都是工程问题：

- 任务拆分
- 记忆设计
- 检索优化
- 幻觉控制
- 动态更新
- 多 Agent 通信
- 协议选型
- 网关治理

说明 AI 面试已经逐步从“模型知识问答”转向“系统设计 + 工程实践”。

## 五、建议的学习顺序

如果按吸收效率来排，推荐顺序可以是：

1. **先学基础边界**
   - Agent 是什么
   - RAG 是什么
   - Function Calling / MCP / Skill 是什么

2. **再学核心链路**
   - Agent：感知、规划、行动、记忆
   - RAG：切块、索引、召回、重排、生成
   - 工具调用：协议、调用链、执行链路

3. **再看架构设计**
   - 单 Agent / 多 Agent
   - 向量检索 / 混合检索 / 图增强
   - FC / MCP / A2A 选型

4. **最后补工程问题**
   - 性能瓶颈
   - 幻觉治理
   - 评估体系
   - 动态更新
   - 通信协议
   - 网关治理

## 六、面试准备建议

基于这页的结构，准备面试时建议重点做到：

- 每个概念都能说出 **定义 + 作用 + 边界 + 适用场景**；
- 每个方案都能说出 **优点 + 代价 + 为什么这么选**；
- 每个工程模块都能说出 **怎么做 + 遇到的问题 + 如何优化**；
- 每个系统设计题都尽量结合真实项目经验来答。

一个实用模板是：

> 先讲概念，再讲链路，再讲取舍，最后讲项目中的实际做法与坑点。
