# 📂 AI Agent 长短期记忆存储与应用深度笔记

记忆系统决定了 Agent 是一个“一次性工具”还是一个“懂你的专业助手”。

---

## 1. 短期记忆 (Working Memory)

- **物理存储**：通常驻留在内存中（Python 的 List 或 Redis 缓存）。
- **生命周期**：Session 级别，任务完成或对话关闭后即清空。
- **存储内容**：原始对话历史（Messages）、中间推理步骤（Thoughts）、工具返回结果（Observations）。
- **优化点：结构化工作记忆**。不只是存消息列表，而是按“目标区”、“结论区”、“假设区”分类存储，保持工作台整洁。

---

## 2. 长期记忆 (Long-term Memory)

- **核心技术**：**Embedding + 向量数据库** (如 ChromaDB, Milvus, Pinecone)。
- **检索逻辑**：语义检索。通过计算向量空间的 Cosine Similarity（余弦相似度）找到最相关的记忆。
- **存储粒度**：
    - **❌ 过细**：按句子拆分。导致检索结果碎片化，上下文丢失。
    - **❌ 过粗**：按整篇文档拆分。导致检索结果包含大量噪音，干扰模型。
    - **✅ 推荐**：按“原子知识点”或“一次完整交互（QA 对）”拆分。

---

## 3. 长期记忆的三大分类

1. **语义记忆 (Semantic)**：存储客观事实（如：用户公司地址、常用技术栈）。
2. **情节记忆 (Episodic)**：存储历史事件流（如：上周二处理 OOM 崩溃的详细过程）。
3. **程序记忆 (Procedural)**：存储方法论（如：给该用户写 PR 的标准检查清单）。

---

## 4. 记忆系统的工程闭环

1. **Recall (回忆)**：在接收新任务的第一时间，主动去长期记忆库检索相关背景。
2. **Inject (注入)**：将检索到的记忆作为 `Background Context` 放入 System Prompt。
3. **Execute (执行)**：在短期记忆的维持下，完成当前任务。
4. **Consolidate (固化)**：任务结束后，提炼重要信息，通过 Embedding 模型写入长期记忆库。

---

## 🎯 面试核心要点

- **不要说“存数据库搜索”**：必须明确提到向量检索和 Embedding，这是区分初级和中高级开发者的关键。
- **记忆衰减机制**：提到“时间衰减因子”或“遗忘曲线”。即：相似度高且时间近的记忆优先召回。
- **信噪比控制**：强调“并不是什么都存”，只有重要的结论、偏好和案例才值得写入长期记忆。
