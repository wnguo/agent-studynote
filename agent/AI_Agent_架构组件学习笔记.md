# 🏗️ AI Agent 基本架构与核心组件笔记

## 一、 架构类比：智能体公司
AI Agent 的架构可以类比为一家**公司**的运作模式：
* **LLM (大脑)**：公司的**老板**，负责最终决策。
* **规划模块 (Planning)**：**项目经理**，负责拆解大目标。
* **记忆系统 (Memory)**：**档案室**，负责信息存档与调档。
* **工具系统 (Tools)**：**外包执行团队**，负责真正干活。

---

## 二、 四大核心组件详解

### 1. LLM 核心 (Brain) —— 决策中心
LLM 是 Agent 的核心推进器，所有输入最终都要经过 LLM 理解。
* **System Prompt**：定义 Agent 的“人格”和行为准则（岗位说明书）。
* **模型选型**：
    * **推理能力**：影响任务拆解的正确性。
    * **JSON 稳定性**：决定工具调用的成败。
    * **上下文窗口**：决定 Agent 能“记得”多少步骤。

### 2. 规划模块 (Planning) —— 任务拆解
* **核心技术**：
    * **CoT (思维链)**：让模型写出思考过程（"Let's think step by step"）。
    * **ToT (思维树)**：在每个节点探索多个分支，选最优路径。
* **主流范式**：
    * **Plan-and-Execute**：先生成完整计划再执行，结构清晰。
    * **ReAct**：边执行边规划，灵活性强，能及时止损纠错。

### 3. 记忆系统 (Memory) —— 状态保持
* **短期记忆**：存储在 **Context Window** 中。记录当前任务的中间结果（如搜索到的网页内容）。
* **长期记忆**：利用 **向量数据库** 实现。
    * **语义记忆**：事实性知识。
    * **情景记忆**：具体的历史经历。
    * **程序性记忆**：做事的方法论。
* **管理机制**：**记忆衰减 (Memory Decay)**。给记忆加权重，随着时间推移降低旧记忆的优先级，防止过时信息干扰。

### 4. 工具系统 (Tools) —— 外部交互
* **决策与执行分离**：模型只负责输出“调哪个工具”，代码负责“执行”。
* **工具描述 (Description)**：模型根据描述判断使用时机。描述质量不亚于 Prompt 工程。
* **标准化协议 (MCP)**：Anthropic 提出的模型上下文协议，相当于工具界的“USB-C 接口”。

---

## 三、 运行核心逻辑 (Loop)
1. **感知**：接收指令 + 检索记忆。
2. **规划**：拆解子任务。
3. **行动**：生成工具调用 JSON -> 执行 -> 获得结果。
4. **反思**：结果存入记忆，评估是否完成，决定下一轮行动。

---

## 🎯 面试要点
* **记忆不是简单的上下文**：要区分短期 (Context) 和长期 (Vector DB)。
* **LLM 不干体力活**：它只负责决策，不直接执行 API。
* **闭环是关键**：强调“感知-规划-行动-再感知”的循环机制。
