# 🧠 AI Agent 记忆机制深度笔记

记忆模块是 Agent 展现个性化、连贯性和专业性的基石。

---

## 1. 记忆的四层金字塔架构

| 层次 | 名称 | 存储载体 | 生命周期 | 核心作用 |
| :--- | :--- | :--- | :--- | :--- |
| **第一层** | **感知记忆** | 原始 Prompt | 瞬时 | 接收当前输入的原始信号 |
| **第二层** | **短期记忆** | 上下文窗口 | 单次任务 | 充当“工作台”，维持当前对话连贯性 |
| **第三层** | **长期记忆** | 向量数据库 | 永久持久化 | 存储历史经历（情节）、知识（语义）和流程（程序） |
| **第四层** | **实体记忆** | 关系型数据库 | 永久持久化 | 存储结构化的关键事实（如：用户画像、项目参数） |

---

## 2. 长期记忆的三大分类

- **情节记忆 (Episodic Memory)**：
  - 记录“发生过什么”。例如：“上次在处理 A 接口报错时，通过增加超时时间解决了。”
- **语义记忆 (Semantic Memory)**：
  - 记录“总结出的规律”。例如：“用户对代码简洁性要求极高，不喜欢冗余注释。”
- **程序记忆 (Procedural Memory)**：
  - 记录“怎么做”。例如：部署服务器的标准 SOP 流程。

---

## 3. 工程化挑战：Context 溢出管理

当对话轮次过多导致 Token 达到上限时，常用的管理策略：
1. **滑动窗口 (Sliding Window)**：仅保留最近的 N 轮对话，丢弃旧对话。
2. **对话总结 (Summary)**：使用 LLM 定期对历史对话进行压缩摘要。
3. **向量检索 (RAG for Memory)**：将旧对话存入向量数据库，仅在语义相关时“回滚”加载。

---

## 4. 设计原则：存、取、用的权衡

- **存什么？** 过滤噪音（如中间过程日志），只存具有长期参考价值的结论和偏好。
- **怎么取？** - **主动加载**：任务开始时加载用户画像。
  - **按需调用**：将“检索记忆”封装为 Agent 的工具。
- **难点**：避免“记忆噪音”。存入过多不相关信息会稀释 LLM 的注意力。

---

## 🎯 面试核心要点

- **核心价值**：记忆让 Agent 从单次交互进化为“持续进化的助手”。
- **架构区分**：能清晰区分向量数据库（语义检索）与关系型数据库（结构化存储）在记忆系统中的分工。
- **优化路径**：提到“动态摘要”和“混合检索策略”是高水平开发者的体现。
