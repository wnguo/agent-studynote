# 🤝 多 Agent 协作与动态切换机制深度笔记

多智能体系统（MAS）是解决复杂业务逻辑、突破单模型 Token 上限及长链推理幻觉的终极方案。

---

## 1. 核心协作模式对比

| 模式 | 协作逻辑 | 核心特点 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **层次式 (Hierarchical)** | Manager 调度下属 | 权限分明，Manager 负责分发与汇总 | 大型软件工程、复杂业务审计 |
| **顺序式 (Sequential)** | 线性接力 | 流程固定，A 的输出是 B 的输入 | 自动化流水线（翻译-校对-排版） |
| **协作式 (Joint/Blackboard)** | 共享黑板 | 全体可见，能者认领任务 | 开放式探索、创意头脑风暴 |

---

## 2. 动态切换机制实现

### A. 静态路由 (Static Routing)
- **实现**：基于关键词或意图分类器。
- **优点**：确定性高，响应快。

### B. 动态规划 (Dynamic Orchestration)
- **实现**：由一个 Master Agent 根据当前任务状态，动态决定下一步调用哪个节点。
- **优点**：灵活，能处理意料之外的分支。

### C. 状态退出 (Condition Exit)
- **实现**：定义明确的停止信号（如 `[FINISH]`）或错误信号。当 Agent 触发信号时，控制权移交给下游或回传给 Manager。

---

## 3. 工程挑战与对策

1. **上下文爆炸**：Agent 频繁对话会导致 Prompt 极长。
   - **对策**：使用**消息摘要**或**选择性上下文传递**。
2. **死循环 (Infinite Loops)**：Agent 互推责任。
   - **对策**：设置**最大全局循环次数**及**观察者监控**。
3. **一致性难题**：多个 Agent 对同一个变量的理解产生歧义。
   - **对策**：引入**中心化状态机 (State)**，作为唯一的“真理来源”。

---

## 🎯 面试核心要点

- **为什么不用单 Agent？** 解释单 Agent 在处理长链复杂逻辑时，注意力会分散，幻觉增加。
- **如何保证效率？** 提到**并行执行**的可能性（独立子任务同步跑）。
- **工具推荐**：提到 **LangGraph** 或 **AutoGen**，这说明你关注行业主流的大规模协作方案。
