# 🕵️ AI Agent 反思机制深度笔记

反思机制（Reflection）是提升 Agent 输出质量、减少幻觉、纠正逻辑错误的“闭环系统”。

---

## 1. 反思的核心循环：Self-Refine

反思不是简单的重试（Retry），而是一个结构化的迭代过程：
1. **生成 (Generate)**：Agent 给出初稿。
2. **评估 (Evaluate)**：使用专用的 Critic Prompt，从逻辑、事实、完整性等维度打分或提建议。
3. **改进 (Refine)**：结合“原稿 + 建议”进行定向优化。

---

## 2. 工程实践中的关键设计

### A. 评估 Prompt 的“黄金法则”
- **量化指标**：不要只问“好不好”，要问“是否包含 5 个要点”、“代码是否能运行”。
- **PASS 信号**：必须赋予模型回复“PASS”或“FINISH”的权利，避免无效迭代。

### B. 粒度选择
- **步骤级反思**：适用于强依赖的长链任务（如复杂代码逻辑）。
- **任务级反思**：适用于文案生成、总结报告等全局性任务。

### C. 强制出口 (Hard Stop)
在代码逻辑中，必须硬性规定最大反思轮次（通常为 **2-3 次**），防止 LLM 因陷入细节纠缠而导致死循环。

---

## 3. 方案进阶：从“自我检查”到“交叉评审”

| 方案 | 特点 | 适用场景 |
| :--- | :--- | :--- |
| **自我反思** | 同一个模型自己检讨 | 简单纠错、格式自查 |
| **Critic Agent** | 独立的模型/Agent 找茬 | 高质量要求（如法律、代码评审） |
| **Reflexion** | 将反思结果作为“经验”存储 | 长期学习、避免重复犯错 |

---

## 🎯 面试核心要点

- **反思 vs 重试**：重试是赌运气，反思是**带反馈的逻辑增强**。
- **局限性**：明确指出反思会增加 **Latency (延迟)** 和 **Token 成本**，因此只需在关键节点开启。
- **自洽性陷阱**：提到多 Agent 互评的必要性——同一个模型往往会对自己的错误产生盲点。
