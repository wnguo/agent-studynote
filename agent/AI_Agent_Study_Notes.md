# 🤖 AI Agent（智能体）学习笔记

## 1. 核心定义：什么是 Agent？
**AI Agent** 本质上是一个能够**自主完成目标**的 AI 系统。它与传统大模型（LLM）的本质区别在于“**自主闭环**”。

* **传统大模型 (LLM)：** 像一个“问答机器”。你问它答，知识停留在训练截止期，且只能输出文本，无法直接操作外部世界。
* **AI Agent：** 像一个“行动派”。你给它一个复杂目标，它会自己拆解任务、调用工具、感知结果并不断调整，直到任务完成。

---

## 2. Agent vs 普通大模型 (对比分析)

| 维度 | 普通大模型 (LLM) | AI Agent |
| :--- | :--- | :--- |
| **交互模式** | 一问一答（被动响应） | 任务导向（主动闭环） |
| **行动能力** | 仅限于文本生成 | 可以调用外部工具（API、数据库） |
| **实时性** | 知识冻结在训练期 | 可通过搜索工具获取实时信息 |
| **反馈机制** | 无反馈，生成即结束 | “感知-规划-行动-再感知”的循环 |

---

## 3. Agent 的三大核心组件
一个完整的 Agent 系统由以下三个“器官”组成：

### ① 工具调用 (Tool Use) —— “手脚”
* **功能：** 突破模型只能“说话”的局限。
* **原理：** 模型通过阅读工具说明书，决定调用哪个工具并生成 JSON 参数，由工程代码负责执行。

### ② 记忆机制 (Memory) —— “大脑存储”
* **短期记忆：** 记录当前任务的执行轨迹、中间结果。
* **长期记忆：** 配合向量数据库，存储用户的历史偏好或专业领域知识。

### ③ 规划与反思 (Planning & Reflection) —— “逻辑核心”
* **任务拆解：** 将复杂目标拆解为多个可执行的子任务。
* **自我纠错：** 当执行失败时，Agent 能分析原因并换一种方法重试。

---

## 4. 行业前沿趋势：MCP 与 A2A
随着 Agent 数量增多，行业开始制定统一的通信标准：

* **MCP (Model Context Protocol):** 由 Anthropic 提出，旨在建立 Agent 与工具之间的“通用接口”，让工具可以被任何 Agent 调用。
* **A2A (Agent2Agent):** 由 Google 提出，解决不同功能的 Agent 之间如何互相协作的问题。

---

## 🎯 面试核心要点总结
1. **自主性：** Agent 的灵魂在于它能“自己决定”何时、如何使用工具。
2. **闭环反馈：** 强调“感知-规划-行动-再感知”的循环。
3. **决策与执行分离：** 模型是“大脑”（负责决策），代码环境是“肢体”（负责执行）。
