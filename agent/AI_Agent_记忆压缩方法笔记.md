# 📉 AI Agent 记忆压缩方法深度笔记

记忆压缩是解决 Context Window 溢出、降低推理成本、提升 Agent 响应质量的关键技术手段。

---

## 1. 记忆压缩的四大流派

### A. 滑动窗口 (Sliding Window)
- **机制**：固定保留最近的 $N$ 条消息。
- **场景**：对历史背景要求不高的短对话。
- **致命伤**：容易丢失对话早期的核心约定（如：用户在第一句说的“不要用 Python”）。

### B. 摘要压缩 (Summarization)
- **机制**：将即将滚出窗口的消息总结为一段话。
- **优化**：**层级式摘要 (Hierarchical Summarization)**。
  - 第一层：最近 10 轮原文。
  - 第二层：11-50 轮的详细摘要。
  - 第三层：51 轮以外的极简结论。

### C. 重要性过滤 (Importance Filtering)
- **机制**：通过打分机制（LLM 打分或规则匹配）决定去留。
- **策略**：**观察遮蔽 (Observation Masking)**。并不真正删除，只是在当前 Prompt 中“隐藏”不相关的历史分支。

### D. 结构化抽取 (Structured Extraction)
- **机制**：将对话转化为结构化数据（JSON/SQL）。
- **场景**：特定领域 Agent（如：客服 Agent 抽取订单号、地址、反馈类型）。
- **优势**：信息密度极大，检索极快。

---

## 2. 计算层优化：Prompt Caching

- **背景**：大模型厂商（OpenAI, Anthropic）提供的技术。
- **原理**：命中缓存的前缀 Token 费用极低（通常为 1/10）。
- **与压缩的关系**：
  - **互补**：记忆压缩减少了需要输入的 Token 总数。
  - **加速**：Prompt Caching 让这些 Token 的处理变得既便宜又快。

---

## 3. 工程决策建议

| 需求场景 | 推荐方案 |
| :--- | :--- |
| **低成本/低复杂度** | 滑动窗口 |
| **通用型助手** | 滑动窗口 + 摘要压缩 |
| **垂直领域/业务流程** | 结构化抽取 + 主动存储 |
| **高频交互/长对话** | 必须开启 Prompt Caching |

---

## 🎯 面试核心要点

- **方案全貌**：不要只答滑动窗口，要说出“摘要、过滤、结构化”三个高阶维度。
- **细节权衡**：摘要压缩的代价是细节丢失，结构化抽取的代价是开发成本和通用性。
- **主动压缩 (Proactive)**：提到“在工具返回结果后立即压缩”比“窗口满了才压缩”更具工程深度。
